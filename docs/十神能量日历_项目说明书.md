# 十神能量日历 · 项目说明书

> **项目名称**：十神能量日历（Bazi Daily Energy）  
> **项目性质**：微信小程序（未来扩展至APP）  
> **文档版本**：V1.0  
> **编制日期**：2026年5月13日  
> **目标读者**：项目全体成员（产品、设计、开发、运营、商务）

---

## 一、项目背景

### 1.1 缘起
项目发起人（百城财税创始人）在淘宝购买了「大闷罐子」品牌的**实体定制台历**。该台历基于中国传统干支历法，根据用户八字中的**日柱天干**（甲、乙、丙、丁、戊、己、庚、辛、壬、癸），将每一天的干支映射为针对该日主的**十神**（正官、七杀、正财、偏财、食神、伤官、比肩、劫财、正印、偏印），形成10个版本的实体日历。

实体台历定价169元/年，采用木质展架，具备文化仪式感，但存在明显局限：
- 仅覆盖单一年份，无法回溯或预览多年
- 无社交属性，无法分享
- 企业批量定制成本高（印刷、物流）
- 便利性不足（必须桌面摆放，无法随身携带）

### 1.2 项目目标
将实体台历的核心逻辑**线上化、社交化、商业化**，开发一款基于干支十神文化的「每日精神能量日历」小程序，同时构建**B2B2C企业礼品定制**的商业模式。

## 二、产品定义

### 2.1 一句话定义
> 基于中国传统干支十神文化的「每日精神能量日历」，每天给用户一个积极的心理锚点，不做吉凶判断，只做正向赋能。

### 2.2 核心原则
| 原则 | 说明 |
|------|------|
| **极简输入** | 仅需出生年月日，时辰可选（默认「未知」），30秒内生成专属日历 |
| **十版本量产** | 以日柱天干为唯一变量，10套规则表（甲~癸），算法极简 |
| **正向-only** | 不出现「凶、忌、破、灾、劫」等负面词汇，所有十神均用积极心理学重新包装 |
| **不做喜忌** | 不判断身强身弱，不推全局喜忌，避免标准化争议 |
| **不做刑冲** | 不标注刑冲合害，避免非专业用户产生误解和焦虑 |
| **社交裂变** | 每日生成精美分享卡片，驱动朋友圈/小红书传播 |
| **企业定制** | 支持批量生成带品牌水印的电子日历，作为客户维护礼品 |

### 2.3 与实体台历对比
| 维度 | 实体台历（大闷罐子） | 十神能量日历小程序 |
|------|---------------------|-------------------|
| 形态 | 木质展架，桌面摆放 | 手机端，随时查看 |
| 年份覆盖 | 仅当年 | 万年历，任意年份 |
| 社交属性 | 无 | 核心功能（分享、送礼） |
| 企业定制 | 印刷成本高，无法批量 | 零边际成本，一键群发 |
| 仪式感 | 强（实体触感） | 弱（便利性碾压） |
| 价格 | 169元/年 | 免费基础版 + 轻付费会员 |

## 三、目标用户

### 3.1 主要用户：中小企业主/管理者（35-50岁）
- **身份**：公司创始人、合伙人、销售总监、客户经理
- **痛点**：客户维护成本高、节日送礼同质化、缺乏「有文化感」的触达方式
- **需求**：低成本、高格调、可量化的客户关怀工具
- **场景**：节前群发定制日历；见客户前看今日十神找话题；朋友圈分享今日能量卡片

### 3.2 次要用户：传统文化爱好者（25-40岁）
- **身份**：对命理、国学有兴趣的都市白领、自由职业者
- **痛点**：想学八字但门槛高；市面上的命理APP太「江湖」，不敢分享
- **需求**：轻量化的日常命理陪伴，能发朋友圈的文化消费品
- **场景**：每天早上看今日能量；特殊日子（签约、面试）前查十神；给朋友送生日日历

## 四、商业模式（三层结构）

### 4.1 第一层：C端免费（获客池）
- **功能**：输入生日生成个人专属十神日历，查看当年每日十神、每日正向寄语
- **意图**：零门槛体验，建立「这是我的专属日历」的心理归属；通过「每日分享卡片」实现社交裂变，低成本获客

### 4.2 第二层：C端轻付费（自循环）
- **定价**：年度会员 9.9~19.9元/年
- **权益**：
  - 多年视图（回溯历史、预览未来）
  - 节气精确到时辰的交节时间
  - 月度能量总结报告
  - 更多精美分享卡片模板
- **意图**：价格比实体台历（169元）低一个数量级，年费制保证持续现金流

### 4.3 第三层：B2B2C礼物定制（利润中心）

**场景A：个人送礼**
- 输入对方生日 → 自动生成对方日主版本的精美电子日历卡
- 可定制祝福语、选择皮肤（生日/春节/商务/国风）
- 生成微信小程序卡片或海报发给对方
- **定价**：单张 3.9~9.9元

**场景B：企业批量定制（主攻方向）**
- 企业上传客户名单（姓名+生日），批量生成「XX公司·2027专属能量日历」
- 每份日历带企业品牌水印和Slogan
- 通过短信/微信/邮件群发，客户点击即领取
- 企业后台可查看领取数据（谁领了、谁分享了）
- **定价**：按份收费，0.5~1元/份，1000份起订
- **延伸行业**：保险、美业、健康管理、高端餐饮、茶叶/白酒、财税服务

**场景C：KOC/命理博主分销**
- 给国学博主开通「品牌联名版」，平台抽成

## 五、产品架构与核心页面

### 5.1 底部导航（4个Tab）
| Tab | 名称 | 核心功能 | 商业意图 |
|-----|------|---------|---------|
| ① | 月历 | 当月网格日历，对标实体台历 | 高频打开，建立使用习惯 |
| ② | 今日 | 今日能量卡片，一键生成分享图 | 社交裂变核心，传播获客 |
| ③ | 送礼 | 给朋友/客户生成定制日历 | 变现核心，B2B2C入口 |
| ④ | 我的 | 个人中心、订单、十神知识 | 留存、会员转化、轻教育 |

### 5.2 核心页面流程
```
启动页（Splash）→ 信息录入页（Onboarding）→ 月历页（Monthly）→ 日详情页（Daily Detail）
```

**页面①：启动页**
- 印章风格Logo（朱砂红 #C84A3E）+ App名「十神能量日历」
- 副标题「基于干支十神 · 每日正向能量」
- CTA按钮「开启我的日历」
- 底部提示「输入生日即可生成专属十神日历」

**页面②：信息录入页**
- **我的生日（必填）**：年月日选择器 + 时辰标签选择（未知/子时/丑时…），默认「未知」
- **送给朋友（可选）**：朋友生日输入框，预埋送礼转化路径
- **底部提示**：「不填时辰，默认按日干推算十神」

**页面③：月历页**
- 顶部月柱信息（如「2026丙午马年 · 5月 · 癸巳月 · 正财偏印」）
- 右侧显示用户日主（如「戊土」）及当月大字十神（如「正财」）
- 星期标题行（周末标红）
- 日历网格：每天四行信息（公历日期 / 农历或节日 / 天干+十神 / 地支+十神）
- 交互：左右滑动切月，点击进详情，长按快速生成分享卡片

**页面④：日详情页**
- 顶部：返回 + 当前日期 + 农历信息
- 大卡片：当日干支大字（如「壬午」）+ 天干十神标签 + 地支十神标签
- 今日寄语区：主寄语（粗体）+ 分点建议（细体）
- 底部功能栏（四宫格）：分享卡片 / 保存图片 / 送给朋友 / 我的日历

## 六、技术方案概要

### 6.1 核心算法
| 模块 | 实现方式 | 复杂度 |
|------|---------|--------|
| 公历转干支 | 嵌入寿星天文历算法，精确处理节气换月 | 成熟算法，已有开源实现 |
| 十神映射 | 10套日主规则表硬编码 | 极低 |
| 地支藏干 | 取本气（主气）即可，如子=癸、寅=甲 | 极低 |
| 文案匹配 | 根据「天干十神+地支十神」组合（100种），从文案库匹配 | 低（内容运营重） |
| 分享卡片生成 | 前端Canvas绘制或后端Puppeteer截图，生成9:16竖图 | 中等 |

### 6.2 技术栈建议
| 层级 | 建议方案 |
|------|---------|
| 前端 | 微信小程序原生 / Taro跨端框架 |
| 后端 | Node.js + Express 或 Python + FastAPI |
| 数据库 | MySQL + Redis |
| 历法计算 | Python lunardate 库 或自研JS历法模块 |
| 图片生成 | Puppeteer + HTML模板截图 / html2canvas |
| 支付 | 微信支付JSAPI |
| 企业后台 | PC端独立H5或Web管理后台 |

## 七、内容资产：正向文案库

### 7.1 文案原则
- **不判断吉凶**：只说「适合做什么」，不说「不宜做什么」
- **赋能导向**：即使是传统「忌神」，也要找到积极角度
- **行动建议**：每条文案包含「今日宜……」的具体行动指引
- **口语化**：避免文言词汇，用现代职场/生活语言
- **可分享性**：控制在20~40字，适合朋友圈卡片展示

### 7.2 文案规模
- **V1.0**：10个十神 × 10条 = **100条基础文案**
- **扩展计划**：
  - V1.1：增加「十神+节气」组合文案
  - V1.2：增加「十神+节日」组合文案
  - V2.0：引入用户反馈闭环，根据「标记事件」数据优化匹配准确度

### 7.3 十神文案速览
| 十神 | 核心能量关键词 |
|------|--------------|
| 正官 | 秩序、贵人、责任 |
| 七杀 | 魄力、突破、攻坚 |
| 正财 | 踏实、收获、稳健 |
| 偏财 | 机遇、灵感、社交 |
| 食神 | 享受、表达、滋养 |
| 伤官 | 创新、破局、锐气 |
| 比肩 | 团队、协作、并肩 |
| 劫财 | 冲刺、争取、拼搏 |
| 正印 | 学习、贵人、蓄力 |
| 偏印 | 洞察、直觉、深度 |

## 九、运营与增长策略

### 9.1 增长飞轮
```
用户生成分享卡片 → 发朋友圈/小红书 → 朋友看到好奇扫码 → 新用户体验 → 生成自己的卡片 → 继续分享
```

### 9.2 三条运营线
| 线路 | 策略 | 目标 |
|------|------|------|
| 社交裂变 | 每日能量卡片自带分享属性；送日历给3位好友双方各得会员 | C端自增长 |
| 企业直销 | 先用百城财税做第一个案例，包装成「企业客户维护新工具」，向其他中小企业销售 | B端现金流 |
| 内容种草 | 短视频：每日十神解读（30秒口播）；小红书：晒「收到的能量日历礼物」 | 品牌认知 |

### 9.3 与百城财税的协同
- 百城财税（2000+企业客户）是第一个种子用户和第一个企业客户
- 向客户发送「百城财税·2027专属能量日历」，作为节日客户维护礼
- 客户领取后自然进入小程序，部分转化为个人用户或会员
- 与「易学顾问」Agent联动：Agent负责深度命理咨询，小程序负责日常轻陪伴

## 十、合规要点

1. **避免敏感词**：小程序审核中，避免「算命、改运、破财消灾、风水」等词汇。产品定位应为「中国传统历法工具」、「干支文化日历」。
2. **免责声明**：所有内容基于传统文化算法，仅供日程参考，不作为决策依据。
3. **不涉及时辰八字算命**：只展示日柱十神，不涉及完整四柱排盘、大运流年推算，降低监管风险。
4. **用户数据**：仅存储年月日（可选时辰），不存储具体出生地点等敏感信息。

## 十一、开发路线图（Roadmap）

| 阶段 | 周期 | 目标 | 核心功能 |
|------|------|------|---------|
| **MVP** | 6~8周 | 跑通核心闭环 | 启动页、录入页、月历页、日详情页、分享卡片生成、个人会员支付 |
| **验证期** | 4周 | 验证企业定制需求 | 企业批量定制H5后台、数据看板、按份计费 |
| **扩张期** | 持续 | 规模化增长 | APP版、多年视图、关系日历、时辰级运势、KOC分销系统 |

### MVP阶段必须验证的假设
1. 用户是否愿意每天打开看十神日历？（留存率）
2. 分享卡片是否真的能带来裂变？（分享率、扫码转化率）
3. 企业客户是否愿意为「电子日历礼品」付费？（B端付费意愿）
4. 正向文案是否让用户感到「被赋能」而非「被忽悠」？（NPS评分）
